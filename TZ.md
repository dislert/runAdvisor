Техническое задание на проект: «Веб-сервис анализа метеорологических условий для любителей бега с целью определения благоприятных погодных условий для уличных тренировок»

1. Цель проекта:
Создание веб-сервиса, который помогает любителям бега принимать решение о проведении уличной тренировки на основе анализа текущих метеорологических условий.
Система автоматически получает погодные данные из внешнего источника, анализирует ключевые параметры и формирует рекомендацию о благоприятности условий для бега.

2. Целевая аудитория и роли пользователей
Целевая аудитория:
Любители бега, которые тренируются на улице и учитывают погодные условия.

Роли пользователей:
Гость — может выбрать город и получить рекомендацию о пригодности текущих погодных условий для уличной пробежки.
Администратор — управляет справочными данными (города), просматривает историю погодных запросов и результатов анализа через административную панель Django.


3. Модели данных
City (Город): Хранение информации о населённых пунктах, для которых доступен анализ погоды.
Поля: name, latitude, longitude.

WeatherSnapshot (Погодные данные): Снимок текущих метеорологических условий для выбранного города.
Поля: city (связь ForeignKey), temperature, wind_speed, humidity, precipitation, timestamp.

RunRecommendation (Рекомендация): Результат анализа погодных условий для уличной тренировки.
Поля: weather (связь OneToOneField), comfort_score, is_good_for_run, explanation.

4. Ключевой функционал (User Stories)
Выбор города:
Пользователь выбирает город из предложенного списка на сайте.

Получение данных:
Система автоматически запрашивает текущие метеорологические данные для выбранного города через внешний API.

Анализ условий:
Система анализирует температуру, скорость ветра, влажность и осадки, рассчитывает итоговый показатель комфортности.

Формирование рекомендации:
На основе анализа система определяет, являются ли текущие погодные условия благоприятными, допустимыми или неблагоприятными для уличной тренировки.

Отображение результата:
Пользователь получает рекомендацию и краткое текстовое объяснение причин данного результата.

5. Внешние интеграции / Аналитика
Внешние интеграции:
Используется внешний метеорологический API - OpenWeatherMap API для получения актуальных погодных данных.

Аналитика: 
Реализация логики расчёта показателя комфортности погодных условий на основе заданных пороговых значений параметров.
